Синхронизируй агентов из проекта-источника.

Аргументы: $ARGUMENTS — путь к проекту-источнику (например: `../other-project`) или URL GitHub репозитория.

## Действие

Обновляет файлы агентов из другого проекта, сохраняя локальные кастомизации.

### Шаг 1: Определить источник

**Если аргумент — локальный путь:**
```bash
ls {path}/.claude/agents/
ls {path}/.claude/commands/
```

**Если аргумент — GitHub URL или `owner/repo`:**
```bash
gh api repos/{owner}/{repo}/contents/.claude/agents --jq '.[].name'
```

**Если аргумент не указан** — спросить у пользователя источник и остановиться.

### Шаг 2: Показать план синхронизации

Сравнить файлы источника и текущего проекта. Вывести:

```
═══ REFRESH PLAN ═══

Агенты (.claude/agents/):
  ✅ analysis-lead.md — обновить (источник новее)
  ✅ dev-lead.md — обновить
  ⏭️ qa-lead.md — пропустить (локальная версия новее)
  ➕ new-agent.md — добавить (новый в источнике)

Команды (.claude/commands/):
  ✅ feature.md — обновить
  ⏭️ analyze.md — пропустить (одинаковые)

Продолжить? (Нажми Enter или опиши что пропустить)
```

Запросить подтверждение перед копированием.

### Шаг 3: Синхронизация

**Из локального пути:**
```bash
cp {path}/.claude/agents/{agent}.md .claude/agents/{agent}.md
```

**Из GitHub:**
```bash
gh api repos/{owner}/{repo}/contents/.claude/agents/{agent}.md \
  --jq '.content' | base64 -d > .claude/agents/{agent}.md
```

Пропустить файлы помеченные как "локальная версия новее" или явно исключённые.

**Никогда не перезаписывать:**
- `.claude/memory/` — локальное состояние проекта
- `.claude/settings.json` — локальные настройки
- `CLAUDE.md` — проектные инструкции

### Шаг 4: Итоговый отчёт

```
═══ REFRESH COMPLETE ═══

Обновлено агентов:  N
Добавлено агентов:  N
Пропущено:          N
Не изменялись:      N

Источник: {path/repo}
```
