# Architecture: Feature #11 — Убрать инпут даты следующей оплаты

## ADR-1: Удалить disabled DatePicker из view_credit.php

**Context:** В форме платежа `views/credit/view_credit.php` отображается DatePicker "Növbəti ödəniş" (Следующая оплата) в строке ежемесячного платежа. Поле уже имеет атрибут `disabled`, данные в него не вводятся. Дата следующей оплаты автоматически пересчитывается в `Payment::afterSave()` / `Payment::afterDelete()` через `Credit::recalculateNextPaymentDate()`.

**Decision:** Удалить `<td>` с DatePicker целиком из HTML-разметки. JS-логика не требуется — поле не используется ни фронтендом, ни бэкендом. Никаких изменений в модели, контроллере или БД не нужно.

**Consequences:** Форма станет чище. Менеджер не отвлекается на лишний элемент. Регрессий нет: логика пересчёта даты живёт в модели и не зависит от этого поля.
